## Why it matters
All security-relevant identifiers are generated with `math/rand/v2`, a
non-cryptographic PRNG. These ids gate access to anonymity primitives:
- a **cid** is the public deep-link handle; knowing a victim's cid lets you open the
  anonymous-message flow to them (`/start <cid>` -> startConnect).
- a **chevaletid** is the per-user pseudonymous key embedded (obfuscated) in every
  token; it is also checked for uniqueness against cids.
- a **report id** is the "tracking code" a reporter is given.
- the **error tracking code** is shown to users.
Predictable/guessable ids enable enumeration and, combined with S-0002's keyless
tokens, forgery.

## Evidence
- `internal/encoder/encoder.go:22` imports `math/rand/v2`; used by:
  - `GenerateCID` (`encoder.go:162-174`) and `GenerateChevaletID` (`encoder.go:180-183`),
  - `EncodeChevaletID` key + patch (`encoder.go:77,93`).
- `internal/bot/callbacks.go:376` report id = `encoder.GenerateCID(20)`.
- `internal/bot/bot.go:142` error code = `encoder.GenerateCID(8)`.
- math/rand/v2 has no seeding call (it is auto-seeded), but it is still a
  non-cryptographic generator and its output stream is not collision/forgery
  resistant the way crypto/rand is.

## Proof / abuse case (defensive)
- cid space mitigates raw brute force somewhat (base57, length 10+2), but cids are
  *enumerable* targets: there is no rate limit on `/start <cid>` (see S-0004), so an
  attacker can probe the cid space to discover live links and message their owners
  anonymously. Non-crypto randomness narrows the practical search and makes
  generated values more predictable than assumed.
- An 8-char report/error code from a predictable PRNG is guessable enough that an
  attacker could assert/forge a plausible "tracking code" to an admin.

## Remediation (concrete)
- Generate all ids from `crypto/rand` (e.g. read random bytes and map to the base57
  alphabet, rejecting modulo bias). This is independent of the Python-parity
  decode contract: only generation needs to change; `DecodeChevaletID` is
  unaffected.
- Combine with rate limiting (S-0004) on `/start <cid>` to defeat enumeration.
