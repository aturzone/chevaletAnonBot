## Why it matters
Several pure, dependency-free functions in internal/bot encode subtle parity rules from the Python
original and are completely untested:
- `findCID` (autoreply.go:86-92) and `findAuthorLink` (autoreply.go:97-109) search a LOWER-cased
  haystack but slice the cid out of the ORIGINAL text to preserve case (the Python `match.span(1)`
  trick). An off-by-one in `loc[2]:loc[3]` would extract the wrong substring and resolve the message
  to the wrong (or no) target â€” i.e. an anonymous reply going to the wrong person.
- `chunkString` / `truncate` (helpers.go:72-98) split on rune boundaries; the error-report pager
  (bot.go:160) and onError summary depend on never cutting a multi-byte Persian character in half.
- `fmtText` (helpers.go:50-55) replaces each `%s` once, left-to-right, and is the parity-safe stand-in
  for Python `%`-formatting against templates that may contain a stray literal `%` or user-supplied
  values (names/tags) â€” the whole point is to NOT break on those, which is exactly the kind of thing a
  test should lock down.
- `DecodeChevaletID` is golden-tested, but its consumers' handling of `ok=false` (handleCIDOrChID
  fallback to treating the token as a cid, cidchid.go:19-29) is not.

These are the cheapest possible tests (no DB, no network, no gotgbot) and they guard exactly the
"who does this anonymous message reach" correctness that matters for 10k users.
